Exchange

In this section I will be exchanging files and their hash values with Tom. The file that I gave to Tom was the PDF for the book jQuery in Action and I provided him the hash values for the three algorithms that I used which are outlined below.

6e7abe112a72b0f2ed3ca8e9733b1801 *jQuery in Action.pdf
c846224851259074bbd1105abca434c58a3bad87 ?SHA1*jQuery in Action.pdf
a92fcf94 ?ADLER32*jQuery in Action.pdf

But I changed the value for the Adler32 hash so it should show up as an error when Tom tests he integrity of the hash values. It should be the following hash value.

a92fcf96 ?ADLER32*jQuery in Action.pdf

Tom's File & Hash Values

The file that I received off Tom was this image of the Boyne Valley Logo. I also received the three hash values that Tom had created using FSUM. These hash values using the same algorithms as what I had been using in my previous tests.

d1087fb1259ed9d9106b4e7976c2c603 *boyne.jpg
a8cabc063556d5b38983ff7314252ba6f03b ?SHA1*boyne.jpg

Firstly I put these into a file named TomsHashValues.txt and then went checked the integrity of the image using the FSUM check option. The screenshot below shows the command and the results of the check.

FSUM was not recognising the SHA-1 hash value so I decided to recreate the hash value for the SHA-1 and below is the hash value that was calculated.
